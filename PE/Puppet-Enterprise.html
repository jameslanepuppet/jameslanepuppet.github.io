We would recommend setting up a live demo environment for yourself to explore Puppet Enterprise and its capabilities. There are some key considerations when doing this listed below but more detailed information can be found in the Puppet docs:

Choosing an architecture

For this environment we would recommend the reference architecture called standard because it supports the needs of the majority of Puppet Enterprise customer's needs.  It can typically manage at least 4,000 nodes with the default 30 minutes check-in interval.  It has an optional HA (high availability) node that should have specs matching the PE Master node.

Installing Puppet Enterprise

The first thing to do is verify that you can install Puppet Enterprise on your chosen Operating system. You can do this by following the link below:

https://puppet.com/docs/pe/2019.8/supported_operating_systems.html#supported-operating-systems


You can find the installation packages for both Puppet Enterprise Primary Server and Puppet Agents via the link below:

https://puppet.com/try-puppet/puppet-enterprise/download/
